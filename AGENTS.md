## 核心技术栈
*   **Frontend:** HTML5
*   **Styling:** Tailwind CSS
*   **Environment:** Static HTML / Browser-based

---

## 执行约束 (Critical Rules)

### 1. 本地预览服务控制
*   **禁止自动启动**：除非用户明确发出类似“启动预览”、“运行服务”或“start dev server”的指令，否则严禁自动启动任何本地预览服务（如 Live Server, Vite, Webpack 等）。
*   **默认行为**：仅生成、修改或提供代码内容，保持静默更新。

### 2. 环境启动约束
*   **Node.js 限制**：严禁在仅调整 HTML 结构或 Tailwind 样式时启动 Node.js 环境。
*   **触发条件**：只有当涉及到必须依赖 Node.js 的逻辑调整（如：安装新的 NPM 包、修改 `tailwind.config.js` 配置文件、或编写复杂的后端 JavaScript 逻辑）时，方可调用或建议启动 Node.js。

### 3. 代码实现规范
*   **轻量化引入**：默认通过项目内本地包引入 Tailwind，避免复杂的构建流程：
    ```html
    <script src="../assets/vendor/tailwind-browser@4.js"></script>
*  **添加代码注释**：对完整的代码模块需添加适当的中文代码注释

### 4. 代码编辑约束
*   **禁止删除已注释代码**：除非用户明确发出类似”清空注释代码“等指令，否则严禁自动删除注释代码

### 5. 设计规范约束
* **优先采用通用组件**：在创建页面或者调整页面时，优先考虑调用已封装的通用组件
* **优先采用公用的业务模块**：在创建页面或调整页面时，相同的业务内容则优先考虑调用已封装的业务模块
* **统一移动端字体规范**：
  * 页面默认字体使用系统默认无衬线字体
  * 页面主标题建议使用 `18px-20px`，字重 `500/700`
  * 模块标题建议使用 `16px-18px`，字重 `500`
  * 正文内容建议使用 `14px-16px`
  * 辅助说明、标签、时间等弱化信息建议使用 `12px-13px`
  * 页面内字体层级不宜过多，建议控制在 `3-4` 个字号层级以内
* **统一移动端颜色规范**：
  * 页面主文字颜色建议使用深色系，推荐 `#333333`
  * 次级说明文字建议使用 `#666666`
  * 弱化提示信息建议使用 `#999999`
  * 页面背景建议使用浅色系，避免大面积高饱和颜色
  * 重点状态、异常状态需通过统一状态色表达，避免同类状态使用多种颜色
  * 单页面主色不宜过多，建议控制在 `1-2` 个主色体系内，保证页面一致性与专业感
